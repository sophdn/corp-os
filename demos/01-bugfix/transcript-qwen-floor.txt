    -provider openai -model-url http://localhost:8081/v1 \
        -model Qwen2.5-32B-Instruct-Q4_K_M.gguf \
    -mid-provider "" -strong-provider "" \        # single-tier: local Qwen only

    -prompt "The Go test in the ./calc package is failing. Find and fix the bug
             in ./calc/calc.go so that 'go test ./calc/' passes. Do not modify

The cheapest capable rung carried the mechanical fix for free; routing the

NOTE (a real gotcha, worth stating): corpos's fs surface operates on the
process's working directory, while -verify-dir scopes the build-test gate. If
you pass -verify-dir X but run corpos from a different directory, the worker
edits one tree while the gate verifies another and the run can never converge.
Run corpos from inside the repo you point -verify-dir at.
